# Contributing to Online Book Store

First of all, thank you for taking the time to contribute to this project. Contributions, whether in the form of bug reports, improvements, or new features, are highly appreciated.

This project is actively maintained and still evolving. The goal is to gradually improve stability, usability, and code quality while keeping the system simple and understandable.

---

## How You Can Contribute

You are welcome to contribute in the following ways:

- Reporting bugs or unexpected behavior
- Suggesting new features or improvements
- Improving UI or user experience
- Enhancing performance
- Fixing existing issues
- Improving code quality and readability

Even small contributions, such as fixing a minor bug or improving documentation, are valuable.

---

## Reporting Issues

If you find a bug or problem in the application:

1. Open a new issue in the repository.
2. Clearly describe the problem.
3. Include steps to reproduce the issue if possible.
4. Add screenshots or logs when relevant.

Before creating a new issue, please check existing issues to avoid duplicates.

This helps maintain clarity and prevents duplicate reports for already known problems.

---

## Pull Request Process

All changes must go through a pull request.

- Fork the repository
- Create a feature or fix branch from `main`
- Make your changes
- Submit a pull request

All pull requests will be reviewed by the maintainer before merging.

Only approved changes will be merged into the main branch.

---

## Branch Strategy

The project follows a simple structure:

- `main` → Stable production-ready code
- `dev` → Development branch for testing and integration

All new contributions should be based on the `dev` branch unless otherwise specified.

After review and testing, changes will be merged into `main`.

---

## Code Style Guidelines

Please follow the existing code style used in the project.

Key expectations:

- Use clear and meaningful variable and method names
- Follow standard C# naming conventions (PascalCase for methods/classes, camelCase for variables)
- Avoid unclear abbreviations
- Keep code readable and maintainable
- Do not introduce unnecessary complexity

The goal is to keep the codebase easy to understand for future contributors and maintainers.

---

## Review Process

All contributions will be manually reviewed.

- Code quality and readability will be checked
- Functionality will be tested before merging
- Changes may be requested before approval

Please be open to feedback and improvements during the review process.

---

## Important Notes

- Always ensure your changes do not break existing functionality
- Try to test your changes before submitting a pull request
- If you are unsure about something, feel free to open an issue first and discuss it

---

## Final Words

This project is intended to grow gradually with community input and continuous improvements. Whether you are fixing a bug or suggesting an idea, your contribution is valuable.

Thank you for helping improve Online Book Store.
